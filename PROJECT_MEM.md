# 🧠 SYSTEM PROMPT & PROJECT MEMORY: HYBRID ORACLE-MANAGER ENGINE

**Core Objective:** Develop, train, and forward-test a hybrid Algorithmic Trading Engine for Gold (XAUUSD) that decouples Pattern Recognition from Risk Management. It uses a Supervised Deep Learning "Oracle" to predict structural Take-Profit probabilities, and a Continuous Reinforcement Learning "Manager" to dynamically size positions and set volatility-based brackets.

## 🏛️ The "Oracle-Manager" Architecture

### 1. The Oracle (Supervised Deep Sequence Model)
* **Goal:** Eliminate the RL "Credit Assignment Problem" by predicting deterministic mathematical outcomes based on structural zones and DXY momentum.
* **Network:** PyTorch LSTM.
* **Input Space:** A rolling 30-candle window of strictly stationary features (Price-agnostic % returns and distances to structural S&R / 4H Wick Zones).
* **Target Output:** Multiclass Softmax `[P_Hold, P_Long, P_Short]`. 
* **Training Logic:** Trained via Weighted Cross-Entropy Loss on historical data. Labels (0, 1, 2) are generated by looking ahead to see if a strict 1:2 Risk/Reward bracket was hit before a Stop Loss.

### 2. The Manager (Continuous RL Agent)
* **Goal:** Optimize the portfolio equity curve by managing capital deployment based on the Oracle's confidence.
* **Network:** Stable-Baselines3 `SAC` (Soft Actor-Critic).
* **Observation Space:** Current Account Balance, Drawdown %, and the Oracle's live probability feed (`[0.10, 0.85, 0.05]`).
* **Action Space:** Continuous `Box` space outputting two values:
  1. `Position_Size` (0.0 to 1.0 -> 0% to 100% of max allowed risk).
  2. `ATR_Multiplier` (-1.0 to 1.0 -> dynamically mapping to 1.0x to 3.0x ATR brackets).

## 🗄️ Critical Architecture Constraints
1. **Absolute Stationarity:** The Oracle must never see raw prices. Standardize all features via Z-score scaling. The training Mean/Std must be exported and loaded by the live environment to prevent distribution shift blindness.
2. **Zero Look-Ahead Bias:** The environment must strictly calculate trailing metrics. Forward-filling or back-filling volatility (ATR) will destroy the integrity of the SAC Manager's training loop.
